# 03. 개발 규칙과 구현 가이드

## 화면 추가 순서

1. 화면 ID와 API 계약을 확인한다.
2. `apps/pages/<feature>/`에 페이지를 만든다.
3. `api/<feature>.js`에 요청 함수를 추가한다.
4. `AdminApp.jsx`의 메뉴와 라우트를 연결한다.
5. loading, empty, error, success 상태를 모두 구현한다.
6. lint/build와 수동 smoke test를 통과시킨다.

## 컴포넌트 규칙

- 파일명은 `PascalCase.jsx`, 함수명도 `PascalCase`를 사용한다.
- 한 컴포넌트가 200줄을 넘거나 두 역할을 가지면 분리한다.
- 버튼은 진행 중 상태와 중복 클릭 방지를 제공한다.
- 주문 상태와 품절 상태는 문자열을 화면마다 새로 만들지 말고 상수로 관리한다.

## UI 상태 최소 기준

| 상태 | 예시 |
| --- | --- |
| loading | 주문 목록 스켈레톤 또는 진행 표시 |
| empty | 조회 조건에 맞는 주문 없음 |
| error | 재시도 버튼과 사용자용 오류 문구 |
| success | 목록/상세/토스트로 결과 확인 |

## 접근성 점검

- 클릭 가능한 요소는 `button` 또는 `a`를 사용한다.
- 아이콘 버튼에는 `aria-label`을 제공한다.
- 색만으로 주문 상태·오류를 구분하지 않는다.
- 키보드 Tab으로 모든 조작 요소에 도달할 수 있어야 한다.
